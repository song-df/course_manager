# 智联学习云 — 自动化产品运营与市场推广方案

> **目标**：在最小化人工参与的前提下，实现用户规模化增长与长期留存。
> **制定日期**：2026-06-11
> **当前基线**：100门课程 | 22,647个视频 | 10大系列 | 0分析系统 | 0SEO | 0自动化运营

---

## 一、现状诊断

### 1.1 产品优势
| 维度 | 现状 |
|------|------|
| 内容规模 | 100门课、22,647个视频，覆盖面广（AI/Java/Python/前端/大数据/算法/职业素养） |
| 技术架构 | 轻量、稳定，视频流支持 Range 分段加载，体验流畅 |
| 安全可控 | 邀请码注册制，首视频免费预览，降低了盗用风险 |
| 用户体验 | 深色主题、响应式设计、章节导航、播放进度追踪 |

### 1.2 关键短板（影响增长与留存）
| 短板 | 影响 |
|------|------|
| **零 SEO** | 搜索引擎完全无法收录课程内容，自然流量为 0 |
| **零数据分析** | 不知道用户从哪来、看了什么、在哪流失 |
| **零自动化触达** | 用户来了就走了，没有任何召回机制 |
| **无支付闭环** | 用户想付费也无法完成，转化路径断裂 |
| **无内容分发** | 优质内容仅限站内，无社交媒体/搜索引擎分发 |
| **无分享机制** | 用户无法一键分享课程给他人，缺乏病毒传播路径 |
| **纯前端进度** | 看课记录存 localStorage，换设备/清缓存即丢失 |

### 1.3 增长瓶颈本质
```
当前漏斗：
  未知用户 → (?) → 访问网站 → 浏览课程 → 观看首视频 → 注册(需邀请码) → 观看更多视频 → (?)
  
核心断裂点：
  ① 获客端：无搜索流量入口，无社交分发
  ② 转化端：邀请码获取路径不透明，无支付对接
  ③ 留存端：无触达手段，无持续价值感知
  ④ 数据端：全链路黑盒，无法优化
```

## 三、第一阶段：基础设施建设（第1-2周）

### 3.1 数据采集与分析系统（最优先）

> **目标**：让每一个用户行为可追踪、可衡量、可优化

#### 实施内容：

**A. 接入轻量级自建分析（推荐 Umami 或 Plausible）**
- 隐私友好，无需 cookie banner
- 部署方式：Docker 一键部署在云主机
- 接入方式：在 4 个 HTML 页面 `<head>` 中插入一行 `<script>`
- 自动采集：PV、UV、访问来源、设备、浏览器、停留时长、跳出率
- 成本：开源免费，服务器已有

**B. 自建关键事件追踪（server.py 增加轻量埋点端点）**
```python
# 新增 /api/track 端点，接收前端事件
# 记录到 SQLite，按日聚合
```
需追踪的核心事件：
| 事件 | 含义 | 自动化用途 |
|------|------|------------|
| `page_view` | 页面浏览 | 热门内容分析 |
| `series_browse` | 系列浏览 | 内容偏好画像 |
| `course_click` | 课程点击 | 推荐算法输入 |
| `video_play` | 视频播放 | 完播率分析 |
| `video_complete` | 视频看完 | 学习进度 |
| `register_start` | 开始注册 | 转化漏斗 |
| `register_complete` | 注册成功 | 转化率 |
| `share_click` | 点击分享 | 病毒系数 |
| `search_query` | 站内搜索 | 内容缺口 |

**C. 简易仪表盘**
- 在 server.py 增加 `/admin/dashboard` 页面（仅内网）
- 每日自动生成关键指标快照
- 指标：日活/周活/月活、新增注册、活跃课程数、完播率、邀请码消耗

### 3.2 SEO 基础设施

> **目标**：让 100 门课程被搜索引擎收录，获取持续免费自然流量

#### 实施内容：

**A. 服务端渲染 (SSR) 化关键页面**

当前所有内容通过 JS 动态渲染，搜索引擎爬虫看到的是空壳。解决方案：

**方案：在 server.py 中增加 SEO 渲染中间件**
```python
# 检测 User-Agent 是否为爬虫（Googlebot/Bingbot/Baiduspider 等）
# 如果是爬虫：返回预渲染的静态 HTML（含完整 meta 标签 + 结构化数据）
# 如果是用户：正常返回 SPA 页面
```

具体实现：
- 为每个系列页面预计算 meta 标签（标题、描述、关键词）
- 为每门课程预计算 meta 标签
- 生成课程结构化数据 (JSON-LD Schema.org `Course` / `VideoObject`)
- 自动生成 Open Graph 标签（用于社交分享预览）

**B. 自动生成 sitemap.xml**
```python
# server.py 新增 /sitemap.xml 端点
# 自动扫描 courses_config.json + series_config.json
# 生成包含所有系列、课程、wiki 页面的 sitemap
# 包含 <lastmod>（基于文件修改时间）
```

**C. robots.txt**
```
# 新增 /robots.txt
User-agent: *
Allow: /
Sitemap: https://aiotedu.cc/sitemap.xml
```

**D. 每个视频页面增加结构化数据**
```html
<!-- 示例：课程页增加 -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Course",
  "name": "深度学习实战",
  "description": "从零开始掌握深度学习核心算法...",
  "provider": {
    "@type": "Organization",
    "name": "智联学习云"
  },
  "numberOfVideos": 45,
  "educationalLevel": "intermediate"
}
</script>
```

### 3.3 支付闭环（支付宝自动化发码）

> **目标**：用户自助完成"付费 → 自动获取邀请码 → 注册 → 观看"全流程，零人工介入

#### 实施方案：

**A. 接入支付宝电脑网站支付**
- 使用支付宝开放平台 API
- 在 `course.html` 视频付费墙弹窗中增加支付按钮
- 支付金额阶梯设计（见 4.2）

**B. 支付回调自动化**
```
用户点击支付 → 支付宝页面 → 支付成功 → 支付宝回调 server.py
→ server.py 调用认证后端 API 生成邀请码
→ 页面自动展示邀请码（同时可选短信/邮件发送）
→ 用户凭邀请码完成注册
```

**C. 增加付费引导页**
- 在 `course.html` 免费预览结束后，展示付费引导弹窗
- 展示内容：课程大纲预览、已学人数、定价方案
- 一键跳转支付宝支付

### 3.4 服务端学习进度同步

> **目标**：用户换设备、清缓存后进度不丢失，为后续个性化推荐打基础

```python
# server.py 新增端点
POST /api/progress/sync   # 同步观看进度（course_id, video_index, progress_seconds）
GET  /api/progress/list    # 获取用户所有课程进度
```

- 已登录用户自动同步
- 匿名用户保留 localStorage 方案
- 为后续"继续学习"功能提供数据

#### C. 百度/Google Search Console 提交
- 自动提交 sitemap 到 Google Search Console API

- 当剩余邀请码 < 10 个时，系统自动通过 API 生成新一批
- 通知管理员（邮件/企微），无需人工干预

## 五、第三阶段：自动化留存体系（第7-10周）

### 5.1 学习进度智能提醒

#### A. 浏览器通知（无需用户联系方式）
- 请求 Web Push Notification 权限

  - 上次学习超过 3 天 → "你已3天没学习了，继续《深度学习实战》？"

  - 新课上线 → "新课上线：《大模型应用开发》"
- 点击通知直接跳转到对应课程页

#### B. 邮件自动化（可选，需接入邮件服务）
- 接入阿里云邮件推送（免费额度 200封/天）
- 自动化邮件序列：

| 触发事件 | 邮件内容 | 延迟 |
|----------|----------|------|
| 注册成功 | 欢迎邮件 + 推荐入门课程 | 即时 |
| 注册后1天未学习 | 提醒 + 一键开始学习链接 | 24h |

| 连续7天学习 | 成就徽章 + 鼓励 | 即时 |

| 新课上线 | 课程推荐（按用户偏好） | 即时 |

- **以上序列全部基于事件自动触发，零人工**

### 5.2 学习激励体系

## 十、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 邀请码泄漏/盗用 | 内容资产损失 | 邀请码绑定设备指纹 + IP，异常使用自动冻结 |
| SEO 沙盒期 | 初期流量增长慢 | 同步推进社交分发和直接引流，不依赖单一渠道 |
| 支付合规问题 | 经营风险 | 确认 ICP 备案包含在线教育/课程销售经营范围 |
| 视频版权争议 | 法律风险 | 确保所有课程内容有合法授权，保留授权记录 |
| 支付宝审核 | 支付接入延迟 | 提前准备营业执照、ICP备案等材料 |
| 服务端性能 | 视频流受分析系统影响 | 埋点日志异步写入，不影响主请求链路 |
